Cloud Equivalents Matrix

"We're on GCP, what do we use?" Use this when the interview pivots from "you said Cassandra" to "we run on Azure". One row per category; one column per cloud + open-source. The mapping is rarely 1:1 — cells include "(closest match, but...)" notes when warranted.

How to read this

  • "Same shape" cells — the cloud product is a near-drop-in replacement.
  • "Closest" cells — the product solves the same problem with different shape (e.g., Firestore vs DynamoDB).
  • "—" cells — the cloud has no native managed equivalent; you self-host the open-source.
  • Cloud-managed offerings of open-source projects (e.g., Memorystore = Redis) are listed alongside the OSS row.

Notes & gotchas

Cosmos DB is many products

Cosmos has SQL, Mongo, Cassandra, Gremlin, and Table APIs. Each "API" is a different shape with different tradeoffs. Don't say "Cosmos DB" without saying which API.

Aurora is not Postgres / MySQL

Aurora is wire-compatible with Postgres or MySQL but the storage layer is different (log-as-DB). Latency profile differs; some extensions don't work.

Pub/Sub vs Kinesis

Both are managed logs. Pub/Sub is global push-or-pull, weakly ordered. Kinesis is regional, strictly per-shard ordered. Pick by ordering need.

App Mesh deprecation

AWS App Mesh end-of-life is announced. New EKS deployments use Istio or Linkerd directly. GCP's CSM and Azure's OSM are similar abstractions.

BigQuery is unique

BigQuery's separation of compute (slots) and storage is not really mirrored by Redshift or Synapse. Snowflake is the closest cross-cloud equivalent.

"Managed Kafka" is multiple things

MSK is your-own-cluster, billed by broker. Confluent Cloud is fully managed and billed by ingress. GCP Managed Kafka is similar to MSK. Event Hubs is a Kafka-API-compatible Azure-native log.

Don't pick a cloud product if you're going multi-cloud

Spanner, BigQuery, DynamoDB, S3 (in places) are sticky. If multi-cloud is on the roadmap, prefer the open-source equivalent.
