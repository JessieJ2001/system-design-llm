Group F — Fanout sister scenarios

Same LC kernel (LC #10 — one writer fans out to N readers). Drastically different designs once a single load-bearing constraint flips.

5 sisters • canonical + 4 candidates

What this topic teaches across its sisters

Every sister in Group F shares the textbook k-way-merge-of-follower-feeds kernel from LeetCode #10 (Design Twitter). None of them survives that textbook structure verbatim, because the load-bearing question — "where does fanout cost live, and how does the trust model decide what's allowed?" — gets answered differently by every premise. Reading the sisters back-to-back trains the constraint-vocabulary the LC kernel hides: access pattern (power-law vs bounded), trust model (server-trusted vs server-blind), connectivity (always-online vs mobile-offline), persistence (ranked inbox vs opaque queue vs ephemeral channel). The same NALSD loop runs through each; only the load-bearing first move differs.

Sisters

#41 Twitter home-timeline fanout (canonical)

Distinguishing factors: read-heavy, power-law (celebrity 200 M followers), trusted server, public posts, ranker on top.
Hallmark iteration: hybrid push/pull split — push for normals, pull for celebrities, replicate celebrity feeds 5×.

F5 WhatsApp / E2E messaging (Phase 3, landed)

Distinguishing factors: server is blind (E2E), bounded recipients (group ≤ 1024), mobile-first offline-tolerant, multi-device.
Hallmark iteration: Signal Protocol E2E (Iter 1) + Sender's Keys for groups (Iter 4) — collapses 1.5 MB upload to 1 KB.

F2 Slack channel messages (planned)

Distinguishing factors: bounded recipients per channel, real-time push, ACL per channel, trusted workplace server.
Anticipated divergence: push-to-channel-members + ACL check at fanout; no celebrity skew; live-tail via SSE.

F3 Enterprise activity feed (planned)

Distinguishing factors: no celebrities, strict ACL, immutable audit log per action, compliance-grade.
Anticipated divergence: append-only event log per tenant + per-event ACL check; SOX-grade retention.

F4 News-website homepage (planned)

Distinguishing factors: no follow graph, popular-content-driven, global cache + edge.
Anticipated divergence: no per-user inbox at all — one ranked global view + per-edge POP cache; refresh-ahead for breaking news.
